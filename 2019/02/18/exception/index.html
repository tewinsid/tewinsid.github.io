java 异常处理流程

异常处理

自定义异常
  • 自定义异常继承运行时异常
  • 运行时异常将会由虚拟机创建，一直抛出到main,并自动调用e.pringStackTace()
  • 重载构造，保留无参构造
  • 一般自定义异常信息，并暴露get方法
  • 运行时异常不会影响调用线程的运行
  • finally 中return会覆盖try或者catch中return
  • try中发生异常 异常后的代码都不会执行包括return
  • 在返回执行表达式前帧栈的局部变量表中存储的变量和引用均是可改变状态
  • 如果finally语句中没有return语句覆盖返回值，那么原来的返回值可能因为finally里的修改而改变也可能不变。
    • 基础数据类型，不会改变。
    • 引用数据类型
      • 修改引用对应堆对象存储会改变
      • 改变指针指向不会生效
    • java可以理解为只有值传递，引用也是值，修改引用会改变堆中的存储，但是重置引用，不会影响堆中的存储
异常流程
  1. try中发生异常，立刻进入catch
  2. 如果catch或者try有return ，限制性finally再 回到 return
  3. finally 在return执行之后，return返回之前执行
